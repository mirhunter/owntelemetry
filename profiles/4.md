# Profile 4 — Advanced Weather Station

Comprehensive weather readings for professional or semi-professional stations on IP or cellular connections. Includes wind, precipitation, UV, and solar radiation in addition to basic atmospheric data.

Location is a fixed property of the station configured on the server — it is not transmitted in packets.

## Packet Types

| Type | Name      | Direction | Description                   |
|------|-----------|-----------|-------------------------------|
| 1    | `reading` | e->s      | Periodic full weather reading |
| 2    | `alert`   | e->s      | Sensor threshold alert        |

## Event Codes

Servers derive human-readable names from the code via the profile schema — no string is transmitted on the wire.

| Code | Name                   | Description                              |
|------|------------------------|------------------------------------------|
| 1    | `high_temperature`     | Temperature exceeded upper threshold     |
| 2    | `low_temperature`      | Temperature fell below lower threshold   |
| 3    | `high_humidity`        | Humidity exceeded upper threshold        |
| 4    | `low_humidity`         | Humidity fell below lower threshold      |
| 5    | `high_wind`            | Sustained wind speed exceeded threshold  |
| 6    | `high_wind_gust`       | Wind gust exceeded threshold             |
| 7    | `high_rainfall_rate`   | Rainfall rate exceeded threshold         |
| 8    | `high_uv`              | UV index exceeded threshold              |
| 9    | `low_pressure`         | Pressure fell below lower threshold      |
| 10   | `high_pressure`        | Pressure exceeded upper threshold        |
| 11   | `low_battery`          | Battery/power level below threshold      |

## Type 1 — Reading

Fields marked **optional** may be omitted from the CBOR packet when not available.

| CBOR Key | Field Name          | Type    | Optional | Description                                                                    |
|----------|---------------------|---------|----------|--------------------------------------------------------------------------------|
| 0        | `profile`           | uint8   |          | Always `4` for this profile                                                    |
| 1        | `id`                | bytes(8)|          | Endpoint device identifier                                                     |
| 2        | `mid`               | uint8   |          | Rolling message ID; server deduplicates on `(mid, timestamp)`                 |
| 3        | `ack`               | bool    |          | `true` if endpoint requests acknowledgment                                     |
| 4        | `type`              | uint8   |          | Always `1` for reading                                                         |
| 5        | `timestamp`         | uint32  |          | Unix timestamp in seconds                                                      |
| 6        | `temperature`       | float32 |          | Air temperature in degrees C                                                   |
| 7        | `humidity`          | float32 |          | Relative humidity as percentage (0.0-100.0)                                   |
| 8        | `pressure`          | float32 |          | Atmospheric pressure in hPa                                                    |
| 9        | `dew_point`         | float32 | yes      | Dew point in degrees C — derivable server-side; omit when not computed        |
| 10       | `wind_speed`        | float32 |          | Sustained wind speed in km/h                                                   |
| 11       | `wind_direction`    | float32 |          | Wind direction in degrees (0-359.9, clockwise from north)                     |
| 12       | `wind_gust`         | float32 |          | Peak wind gust speed in km/h since last reading                                |
| 13       | `rainfall`          | float32 |          | Accumulated rainfall in mm since last reading                                  |
| 14       | `rainfall_rate`     | float32 |          | Current rainfall rate in mm/h                                                  |
| 15       | `uv_index`          | float32 |          | UV index                                                                       |
| 16       | `solar_radiation`   | float32 |          | Solar radiation in W/m2                                                        |
| 17       | `visibility`        | float32 | yes      | Visibility in km — requires optical/laser sensor; omit when not present       |
| 18       | `battery_level`     | float32 |          | Battery/power level as percentage (0.0-100.0)                                 |

### Example

```json
{
  "profile": 4,
  "id": "c2f3e4d5a6b7c8d9",
  "mid": 31,
  "ack": false,
  "type": 1,
  "timestamp": 1622547800,
  "temperature": 21.3,
  "humidity": 58.4,
  "pressure": 1015.6,
  "wind_speed": 14.5,
  "wind_direction": 232.0,
  "wind_gust": 22.1,
  "rainfall": 0.0,
  "rainfall_rate": 0.0,
  "uv_index": 4.2,
  "solar_radiation": 612.0,
  "battery_level": 91.0
}
```

## Type 2 — Alert

The `value` field contains the sensor reading that triggered the alert.

| CBOR Key | Field Name    | Type      | Description                                                    |
|----------|---------------|-----------|----------------------------------------------------------------|
| 0        | `profile`     | uint8     | Always `4` for this profile                                    |
| 1        | `id`          | bytes (8) | Endpoint device identifier                                     |
| 2        | `mid`         | uint8     | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`         | bool      | `true` if endpoint requests acknowledgment                     |
| 4        | `type`        | uint8     | Always `2` for alert                                           |
| 5        | `event_code`  | uint8     | Event code (see above)                                         |
| 6        | `timestamp`   | uint32    | Unix timestamp in seconds                                      |
| 7        | `value`       | float32   | Sensor value that triggered the alert                          |

### Example

```json
{
  "profile": 4,
  "id": "c2f3e4d5a6b7c8d9",
  "mid": 32,
  "ack": true,
  "type": 2,
  "event_code": 6,
  "timestamp": 1622548200,
  "value": 67.3
}
```
