# Profile 8 — Marine

Vessel position and navigation data for boats and small ships. Core fields cover fix state, speed over ground (SOG), and course over ground (COG). Position, heading, depth, and wind data are optional depending on available instruments and fix state.

Location is transmitted in each packet — unlike weather station profiles, vessels are mobile.

## Packet Types

| Type | Name       | Direction | Description                             |
|------|------------|-----------|-----------------------------------------|
| 1    | `position` | e->s      | Periodic vessel position and navigation |
| 2    | `alert`    | e->s      | Navigation or safety alert              |

## Event Codes

| Code | Name             | Description                                         |
|------|------------------|-----------------------------------------------------|
| 1    | `anchor_drag`    | Vessel has moved beyond anchor watch radius         |
| 2    | `man_overboard`  | Man overboard alert triggered                       |
| 3    | `collision_risk` | Proximity alert — another vessel or obstacle nearby |
| 4    | `low_battery`    | Battery/power level below operational threshold     |

---

## Type 1 — Position

`fix` is always transmitted. `lat` and `lon` are omitted when `fix` is `0`. `sog` and `cog` are always required — they are the primary navigation fields and may remain valid from dead reckoning even without GPS. `heading`, `depth`, `wind_speed`, and `wind_direction` require dedicated instruments and are optional.

| CBOR Key | Field Name        | Type      | Optional | Description                                                    |
|----------|-------------------|-----------|----------|----------------------------------------------------------------|
| 0        | `profile`         | uint8     |          | Always `8` for this profile                                    |
| 1        | `id`              | bytes (8) |          | Endpoint device identifier                                     |
| 2        | `mid`             | uint8     |          | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`             | bool      |          | `true` if endpoint requests acknowledgment                     |
| 4        | `type`            | uint8     |          | Always `1` for position                                        |
| 5        | `timestamp`       | uint32    |          | Unix timestamp in seconds                                      |
| 6        | `fix`             | uint8     |          | GPS fix type: `0` no fix, `1` 2D fix, `2` 3D fix             |
| 7        | `lat`             | float32   | yes      | Latitude in decimal degrees (present when fix >= 1)            |
| 8        | `lon`             | float32   | yes      | Longitude in decimal degrees (present when fix >= 1)           |
| 9        | `sog`             | float32   |          | Speed over ground in knots                                     |
| 10       | `cog`             | float32   |          | Course over ground in degrees (0-359.9, clockwise from north)  |
| 11       | `heading`         | float32   | yes      | True heading in degrees (compass/gyro)                         |
| 12       | `depth`           | float32   | yes      | Water depth in meters (from depth sounder)                     |
| 13       | `wind_speed`      | float32   | yes      | Wind speed in knots                                            |
| 14       | `wind_direction`  | float32   | yes      | Wind direction in degrees (true, clockwise from north)         |
| 15       | `battery_voltage` | float32   | yes      | House battery voltage                                          |

### Example

```json
{
  "profile": 8,
  "id": "d4a5b6c7e8f9a0b1",
  "mid": 31,
  "ack": false,
  "type": 1,
  "timestamp": 1622547800,
  "fix": 2,
  "lat": 50.812,
  "lon": -1.134,
  "sog": 5.4,
  "cog": 182.0,
  "heading": 179.0,
  "depth": 8.2,
  "wind_speed": 12.0,
  "wind_direction": 225.0
}
```

## Type 2 — Alert

`lat`/`lon` capture the vessel position at the time of the alert. `value` carries context data where relevant (e.g. distance dragged for `anchor_drag`).

| CBOR Key | Field Name   | Type      | Optional | Description                                                    |
|----------|--------------|-----------|----------|----------------------------------------------------------------|
| 0        | `profile`    | uint8     |          | Always `8` for this profile                                    |
| 1        | `id`         | bytes (8) |          | Endpoint device identifier                                     |
| 2        | `mid`        | uint8     |          | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`        | bool      |          | `true` if endpoint requests acknowledgment                     |
| 4        | `type`       | uint8     |          | Always `2` for alert                                           |
| 5        | `event_code` | uint8     |          | Event code (see above)                                         |
| 6        | `timestamp`  | uint32    |          | Unix timestamp in seconds                                      |
| 7        | `lat`        | float32   | yes      | Latitude at time of alert                                      |
| 8        | `lon`        | float32   | yes      | Longitude at time of alert                                     |
| 9        | `value`      | float32   | yes      | Context value (e.g. metres dragged for anchor_drag)            |

### Example

```json
{
  "profile": 8,
  "id": "d4a5b6c7e8f9a0b1",
  "mid": 32,
  "ack": true,
  "type": 2,
  "event_code": 1,
  "timestamp": 1622549000,
  "lat": 50.809,
  "lon": -1.137,
  "value": 42.0
}
```
