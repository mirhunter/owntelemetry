# Profile 2 — Advanced GPS Telemetry (Fleet Vehicle)

Comprehensive vehicle tracking for cellular-connected fleet devices — cars, vans, and heavy trucks.

## Packet Types

| Type | Name       | Direction | Description                                  |
|------|------------|-----------|----------------------------------------------|
| 1    | `location` | e->s      | Periodic position and vehicle state update   |
| 2    | `event`    | e->s      | Driver behavior event with position context  |

---

## Type 1 — Location

`fix` is always transmitted. `lat`, `lon`, `alt`, and `accuracy` are omitted when the fix state does not support them. `fuel_level`, `engine_temp`, and `battery_voltage` are optional — present only when the hardware supports them.

| CBOR Key | Field Name        | Type      | Optional | Description                                                    |
|----------|-------------------|-----------|----------|----------------------------------------------------------------|
| 0        | `profile`         | uint8     |          | Always `2` for this profile                                    |
| 1        | `id`              | bytes (8) |          | Endpoint device identifier                                     |
| 2        | `mid`             | uint8     |          | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`             | bool      |          | `true` if endpoint requests acknowledgment                     |
| 4        | `type`            | uint8     |          | Always `1` for location                                        |
| 5        | `timestamp`       | uint32    |          | Unix timestamp in seconds                                      |
| 6        | `fix`             | uint8     |          | GPS fix type: `0` no fix, `1` 2D fix, `2` 3D fix             |
| 7        | `lat`             | float32   | yes      | Latitude in decimal degrees (present when fix >= 1)            |
| 8        | `lon`             | float32   | yes      | Longitude in decimal degrees (present when fix >= 1)           |
| 9        | `alt`             | float32   | yes      | Altitude in meters (present when fix = 2)                      |
| 10       | `accuracy`        | float32   | yes      | Horizontal accuracy estimate in meters                         |
| 11       | `speed`           | float32   |          | Speed in km/h                                                  |
| 12       | `heading`         | float32   |          | Direction of travel in degrees (0-359.9, clockwise from north) |
| 13       | `satellites`      | uint8     |          | Number of GPS satellites used in fix                           |
| 14       | `hdop`            | float32   |          | Horizontal dilution of precision (lower is better; <2.0 good) |
| 15       | `ignition`        | bool      |          | `true` if ignition is on                                       |
| 16       | `moving`          | bool      |          | `true` if vehicle is in motion                                 |
| 17       | `odometer`        | uint32    |          | Odometer reading in km                                         |
| 18       | `rpm`             | uint16    |          | Engine RPM                                                     |
| 19       | `fuel_level`      | float32   | yes      | Fuel level as percentage (0.0-100.0)                          |
| 20       | `engine_temp`     | float32   | yes      | Engine coolant temperature in degrees C                        |
| 21       | `battery_voltage` | float32   | yes      | Vehicle battery voltage (typically ~12 V cars, ~24 V trucks)  |

### Example

```json
{
  "profile": 2,
  "id": "a3f1c2b4d5e6f7a8",
  "mid": 17,
  "ack": false,
  "type": 1,
  "timestamp": 1622547800,
  "fix": 2,
  "lat": 45.678,
  "lon": -123.456,
  "alt": 85.0,
  "accuracy": 3.2,
  "speed": 88.5,
  "heading": 274.3,
  "satellites": 9,
  "hdop": 1.2,
  "ignition": true,
  "moving": true,
  "odometer": 142300,
  "rpm": 2100,
  "fuel_level": 63.4,
  "engine_temp": 91.0,
  "battery_voltage": 14.2
}
```

## Event Codes

| Code | Name           | Description                  |
|------|----------------|------------------------------|
| 1    | `harsh_accel`  | Sudden acceleration detected |
| 2    | `harsh_brake`  | Sudden braking detected      |
| 3    | `harsh_corner` | Harsh cornering detected     |

## Type 2 — Event

`fix` is always transmitted. `lat`, `lon`, and `altitude` are omitted when the fix state does not support them.

| CBOR Key | Field Name   | Type      | Optional | Description                                                    |
|----------|--------------|-----------|----------|----------------------------------------------------------------|
| 0        | `profile`    | uint8     |          | Always `2` for this profile                                    |
| 1        | `id`         | bytes (8) |          | Endpoint device identifier                                     |
| 2        | `mid`        | uint8     |          | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`        | bool      |          | `true` if endpoint requests acknowledgment                     |
| 4        | `type`       | uint8     |          | Always `2` for event                                           |
| 5        | `event_code` | uint8     |          | Numeric event code (see above)                                 |
| 6        | `timestamp`  | uint32    |          | Unix timestamp in seconds                                      |
| 7        | `fix`        | uint8     |          | GPS fix type: `0` no fix, `1` 2D fix, `2` 3D fix             |
| 8        | `lat`        | float32   | yes      | Latitude at time of event (present when fix >= 1)              |
| 9        | `lon`        | float32   | yes      | Longitude at time of event (present when fix >= 1)             |
| 10       | `altitude`   | float32   | yes      | Altitude in meters at time of event (present when fix = 2)     |
| 11       | `speed`      | float32   |          | Speed in km/h at time of event                                 |
| 12       | `heading`    | float32   |          | Direction of travel in degrees at time of event                |
| 13       | `signal`     | int8      |          | Cellular signal strength in dBm                                |

### Example

```json
{
  "profile": 2,
  "id": "a3f1c2b4d5e6f7a8",
  "mid": 18,
  "ack": true,
  "type": 2,
  "event_code": 2,
  "timestamp": 1622547923,
  "fix": 2,
  "lat": 45.681,
  "lon": -123.460,
  "altitude": 85.0,
  "speed": 72.1,
  "heading": 271.8,
  "signal": -83
}
```
