# Profile 5 — Asset Tracker

LoRa-optimised battery-powered asset tracking for livestock, equipment, vehicles, and luggage. Designed to be minimal on the wire — battery level and motion state are always reported; position is included when a GPS fix is available.

## Packet Types

| Type | Name       | Direction | Description                         |
|------|------------|-----------|-------------------------------------|
| 1    | `location` | e->s      | Periodic position and motion state  |
| 2    | `alert`    | e->s      | Threshold or state-change event     |

## Event Codes

| Code | Name             | Description                                  |
|------|------------------|----------------------------------------------|
| 1    | `low_battery`    | Battery level below operational threshold    |
| 2    | `motion_start`   | Asset began moving                           |
| 3    | `motion_stop`    | Asset stopped moving                         |
| 4    | `geofence_exit`  | Asset left a server-defined geofence zone    |
| 5    | `geofence_enter` | Asset entered a server-defined geofence zone |

---

## Type 1 — Location

`fix` is always transmitted. `lat` and `lon` are omitted when `fix` is `0`. This allows the endpoint to report battery level and motion state even when GPS is unavailable.

| CBOR Key | Field Name      | Type      | Optional | Description                                                    |
|----------|-----------------|-----------|----------|----------------------------------------------------------------|
| 0        | `profile`       | uint8     |          | Always `5` for this profile                                    |
| 1        | `id`            | bytes (8) |          | Endpoint device identifier                                     |
| 2        | `mid`           | uint8     |          | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`           | bool      |          | `true` if endpoint requests acknowledgment                     |
| 4        | `type`          | uint8     |          | Always `1` for location                                        |
| 5        | `timestamp`     | uint32    |          | Unix timestamp in seconds                                      |
| 6        | `fix`           | uint8     |          | GPS fix type: `0` no fix, `1` 2D fix, `2` 3D fix             |
| 7        | `lat`           | float32   | yes      | Latitude in decimal degrees (present when fix >= 1)            |
| 8        | `lon`           | float32   | yes      | Longitude in decimal degrees (present when fix >= 1)           |
| 9        | `battery_level` | float32   |          | Battery level as percentage (0.0-100.0)                       |
| 10       | `moving`        | bool      |          | `true` if asset is currently in motion                         |
| 11       | `accuracy`      | float32   | yes      | Horizontal accuracy estimate in meters                         |

### Example — fix acquired

```json
{
  "profile": 5,
  "id": "a3f1c2b4d5e6f7a8",
  "mid": 14,
  "ack": false,
  "type": 1,
  "timestamp": 1622547800,
  "fix": 2,
  "lat": 51.503,
  "lon": -0.119,
  "battery_level": 72.0,
  "moving": false
}
```

### Example — no fix

```json
{
  "profile": 5,
  "id": "a3f1c2b4d5e6f7a8",
  "mid": 15,
  "ack": false,
  "type": 1,
  "timestamp": 1622547860,
  "fix": 0,
  "battery_level": 72.0,
  "moving": false
}
```

## Type 2 — Alert

`lat`/`lon` capture the position at the time of the alert when available. `value` carries context data — for `low_battery` this is the battery percentage; for geofence events it may be omitted.

| CBOR Key | Field Name   | Type      | Optional | Description                                                    |
|----------|--------------|-----------|----------|----------------------------------------------------------------|
| 0        | `profile`    | uint8     |          | Always `5` for this profile                                    |
| 1        | `id`         | bytes (8) |          | Endpoint device identifier                                     |
| 2        | `mid`        | uint8     |          | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`        | bool      |          | `true` if endpoint requests acknowledgment                     |
| 4        | `type`       | uint8     |          | Always `2` for alert                                           |
| 5        | `event_code` | uint8     |          | Event code (see above)                                         |
| 6        | `timestamp`  | uint32    |          | Unix timestamp in seconds                                      |
| 7        | `lat`        | float32   | yes      | Latitude at time of alert                                      |
| 8        | `lon`        | float32   | yes      | Longitude at time of alert                                     |
| 9        | `value`      | float32   | yes      | Context value (e.g. battery % for `low_battery`)              |

### Example

```json
{
  "profile": 5,
  "id": "a3f1c2b4d5e6f7a8",
  "mid": 16,
  "ack": true,
  "type": 2,
  "event_code": 1,
  "timestamp": 1622548100,
  "lat": 51.503,
  "lon": -0.119,
  "value": 14.2
}
```
