# Profile 3 — Basic Weather Station

Minimal weather readings suitable for constrained transports (LoRa). Targets simple sensor nodes (e.g. temperature/humidity/pressure module) that report periodically and alert on threshold violations.

Location is a fixed property of the station configured on the server — it is not transmitted in packets.

## Packet Types

| Type | Name      | Direction | Description              |
|------|-----------|-----------|--------------------------|
| 1    | `reading` | e->s      | Periodic weather reading |
| 2    | `alert`   | e->s      | Sensor threshold alert   |

## Event Codes

Servers derive human-readable names from the code via the profile schema — no string is transmitted on the wire.

| Code | Name                | Description                              |
|------|---------------------|------------------------------------------|
| 1    | `high_temperature`  | Temperature exceeded upper threshold     |
| 2    | `low_temperature`   | Temperature fell below lower threshold   |
| 3    | `high_humidity`     | Humidity exceeded upper threshold        |
| 4    | `low_humidity`      | Humidity fell below lower threshold      |
| 5    | `low_pressure`      | Pressure fell below lower threshold      |
| 6    | `high_pressure`     | Pressure exceeded upper threshold        |
| 7    | `low_battery`       | Battery level below operational threshold |

---

## Type 1 — Reading

| CBOR Key | Field Name      | Type      | Description                                                    |
|----------|-----------------|-----------|----------------------------------------------------------------|
| 0        | `profile`       | uint8     | Always `3` for this profile                                    |
| 1        | `id`            | bytes (8) | Endpoint device identifier                                     |
| 2        | `mid`           | uint8     | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`           | bool      | `true` if endpoint requests acknowledgment                     |
| 4        | `type`          | uint8     | Always `1` for reading                                         |
| 5        | `timestamp`     | uint32    | Unix timestamp in seconds                                      |
| 6        | `temperature`   | float32   | Air temperature in degrees C                                   |
| 7        | `humidity`      | float32   | Relative humidity as percentage (0.0-100.0)                   |
| 8        | `pressure`      | float32   | Atmospheric pressure in hPa                                    |
| 9        | `battery_level` | float32   | Battery level as percentage (0.0-100.0)                       |

### Example

```json
{
  "profile": 3,
  "id": "b1e2d3c4f5a6b7c8",
  "mid": 7,
  "ack": false,
  "type": 1,
  "timestamp": 1622547800,
  "temperature": 18.4,
  "humidity": 62.1,
  "pressure": 1013.25,
  "battery_level": 84.0
}
```

## Type 2 — Alert

The `value` field contains the sensor reading that triggered the alert.

| CBOR Key | Field Name    | Type      | Description                                                    |
|----------|---------------|-----------|----------------------------------------------------------------|
| 0        | `profile`     | uint8     | Always `3` for this profile                                    |
| 1        | `id`          | bytes (8) | Endpoint device identifier                                     |
| 2        | `mid`         | uint8     | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`         | bool      | `true` if endpoint requests acknowledgment                     |
| 4        | `type`        | uint8     | Always `2` for alert                                           |
| 5        | `event_code`  | uint8     | Event code (see above)                                         |
| 6        | `timestamp`   | uint32    | Unix timestamp in seconds                                      |
| 7        | `value`       | float32   | Sensor value that triggered the alert                          |

### Example

```json
{
  "profile": 3,
  "id": "b1e2d3c4f5a6b7c8",
  "mid": 8,
  "ack": true,
  "type": 2,
  "event_code": 2,
  "timestamp": 1622548100,
  "value": -3.2
}
```
