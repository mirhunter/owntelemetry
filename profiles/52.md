# Profile 52 — Model Rocket

> **User-defined profile** (range 51-250).

Flight telemetry and recovery tracking for model rockets. Three packet types cover pre/post-flight checks, in-flight sensor data, and discrete flight phase events.

The status packet transmits pad coordinates which serve as the ground reference for AGL altitude calculations and as a recovery search origin if GPS is unavailable during flight.

## CRC

This profile enables CRC-16/CCITT on all packet types. All radio links used in rocketry (LoRa, dedicated telemetry radios, bare RF) benefit from application-level integrity checking. The `crc` field is always the last field in every packet.

## Timestamp Resolution

The `timestamp` field uses **seconds** in the status packet (pre/post flight) and **microseconds** in the flight and event packets. This is a deliberate exception to the default seconds convention — motor burn times, apogee detection, and pyro firing all require sub-millisecond resolution to be meaningful.

The deduplication key `(mid, timestamp)` remains valid: multipath duplicates of the same packet will share the same microsecond timestamp.

## Packet Types

| Type | Name     | Direction | Description                                                    |
|------|----------|-----------|----------------------------------------------------------------|
| 1    | `status` | e->s      | Pre/post-flight check — pad position, pyro continuity, armed  |
| 2    | `flight` | e->s      | High-frequency in-flight telemetry — altitude, velocity, IMU  |
| 3    | `event`  | e->s      | Discrete flight phase event                                    |

## Event Codes

Servers derive human-readable names from the code via the profile schema — no string is transmitted on the wire.

| Code | Name               | Description                           |
|------|--------------------|---------------------------------------|
| 1    | `launch_detected`  | Liftoff detected by accelerometer     |
| 2    | `motor_burnout`    | Thrust phase ended                    |
| 3    | `apogee`           | Peak altitude reached                 |
| 4    | `drogue_deploy`    | Drogue chute deployed                 |
| 5    | `main_deploy`      | Main chute deployed                   |
| 6    | `landing`          | Landing detected                      |

## Type 1 — Status

Sent before arming and after landing. `lat`/`lon` record the pad or landing position; `altitude` is calibrated to ground level (0.0 at pad).

| CBOR Key | Field Name          | Type      | Description                                                     |
|----------|---------------------|-----------|-----------------------------------------------------------------|
| 0        | `profile`           | uint8     | Always `52` for this profile                                    |
| 1        | `id`                | bytes (8) | Endpoint device identifier                                      |
| 2        | `mid`               | uint8     | Rolling message ID                                              |
| 3        | `ack`               | bool      | `true` if endpoint requests acknowledgment                      |
| 4        | `type`              | uint8     | Always `1` for status                                           |
| 5        | `timestamp`         | uint32    | Unix timestamp in seconds                                       |
| 6        | `lat`               | float32   | Latitude in decimal degrees                                     |
| 7        | `lon`               | float32   | Longitude in decimal degrees                                    |
| 8        | `altitude`          | float32   | Altitude in meters (0.0 = ground level at pad)                  |
| 9        | `battery_voltage`   | float32   | Flight computer battery voltage                                 |
| 10       | `armed`             | bool      | `true` if pyro channels are armed                               |
| 11       | `pyro_1`            | bool      | `true` if pyro channel 1 has continuity (apogee / drogue)      |
| 12       | `pyro_2`            | bool      | `true` if pyro channel 2 has continuity (main chute)           |
| 13       | `temperature`       | float32   | Internal temperature in degrees C                               |
| 14       | `rssi`              | int8      | Received signal strength in dBm                                 |
| 15       | `crc`               | uint16    | CRC-16/CCITT (CRC field set to 0x0000 during calculation)      |

### Example

```json
{
  "profile": 52,
  "id": "e5f6a7b8c9d0e1f2",
  "mid": 1,
  "ack": true,
  "type": 1,
  "timestamp": 1622547600,
  "lat": 35.347,
  "lon": -117.808,
  "altitude": 0.0,
  "battery_voltage": 8.34,
  "armed": false,
  "pyro_1": true,
  "pyro_2": true,
  "temperature": 28.1,
  "rssi": -72,
  "crc": 0
}
```

> **Note:** `"crc": 0` is a placeholder. Implementations must compute the correct CRC-16/CCITT value over all CBOR-encoded packet bytes with the CRC field set to `0x0000`, then replace this field with the result before transmitting.

## Type 2 — Flight

High-frequency packet sent during flight. `lat`/`lon` are **optional** — include when GPS is available, omit (not zero) when it is not. `altitude` is AGL (above ground level, relative to pad). Acceleration values are in m/s2; rotation rates in degrees/second.

| CBOR Key | Field Name      | Type    | Optional | Description                                                        |
|----------|-----------------|---------|----------|--------------------------------------------------------------------|
| 0        | `profile`       | uint8   |          | Always `52` for this profile                                       |
| 1        | `id`            | bytes(8)|          | Endpoint device identifier                                         |
| 2        | `mid`           | uint8   |          | Rolling message ID                                                 |
| 3        | `ack`           | bool    |          | `true` if requesting acknowledgment — avoid on high-freq packets   |
| 4        | `type`          | uint8   |          | Always `2` for flight                                              |
| 5        | `timestamp`     | uint64  |          | Unix timestamp in **microseconds**                                 |
| 6        | `altitude`      | float32 |          | Altitude AGL in meters                                             |
| 7        | `velocity`      | float32 |          | Vertical velocity in m/s (positive = ascending)                    |
| 8        | `accel_x`       | float32 |          | Acceleration on X axis in m/s2                                    |
| 9        | `accel_y`       | float32 |          | Acceleration on Y axis in m/s2                                    |
| 10       | `accel_z`       | float32 |          | Acceleration on Z axis in m/s2 (axial / thrust axis)              |
| 11       | `roll`          | float32 |          | Roll rate in degrees/second                                        |
| 12       | `pitch`         | float32 |          | Pitch rate in degrees/second                                       |
| 13       | `yaw`           | float32 |          | Yaw rate in degrees/second                                         |
| 14       | `pressure`      | float32 |          | Atmospheric pressure in hPa (barometric altitude source)           |
| 15       | `temperature`   | float32 |          | Internal temperature in degrees C                                  |
| 16       | `lat`           | float32 | yes      | Latitude in decimal degrees (GPS — omit if no fix)                 |
| 17       | `lon`           | float32 | yes      | Longitude in decimal degrees (GPS — omit if no fix)                |
| 18       | `rssi`          | int8    |          | Received signal strength in dBm                                    |
| 19       | `crc`           | uint16  |          | CRC-16/CCITT (CRC field set to 0x0000 during calculation)         |

### Example

```json
{
  "profile": 52,
  "id": "e5f6a7b8c9d0e1f2",
  "mid": 14,
  "ack": false,
  "type": 2,
  "timestamp": 1622547613000000,
  "altitude": 287.4,
  "velocity": 48.2,
  "accel_x": 0.3,
  "accel_y": 0.1,
  "accel_z": 24.7,
  "roll": 2.1,
  "pitch": 0.4,
  "yaw": 0.2,
  "pressure": 977.3,
  "temperature": 14.6,
  "rssi": -89,
  "crc": 0
}
```

> **Note:** `"crc": 0` is a placeholder. `lat`/`lon` are omitted here (no GPS fix). `ack` should be `false` on high-frequency flight packets to avoid flooding the limited downlink.

## Type 3 — Event

Sent at each discrete flight phase transition. `altitude` and `velocity` capture conditions at the moment of the event.

| CBOR Key | Field Name    | Type    | Description                                                        |
|----------|---------------|---------|--------------------------------------------------------------------|
| 0        | `profile`     | uint8   | Always `52` for this profile                                       |
| 1        | `id`          | bytes(8)| Endpoint device identifier                                         |
| 2        | `mid`         | uint8   | Rolling message ID                                                 |
| 3        | `ack`         | bool    | `true` if endpoint requests acknowledgment                         |
| 4        | `type`        | uint8   | Always `3` for event                                               |
| 5        | `event_code`  | uint8   | Event code (see above)                                             |
| 6        | `timestamp`   | uint64  | Unix timestamp in **microseconds** — precise event timing          |
| 7        | `altitude`    | float32 | Altitude AGL in meters at time of event                            |
| 8        | `velocity`    | float32 | Vertical velocity in m/s at time of event                          |
| 9        | `rssi`        | int8    | Received signal strength in dBm                                    |
| 10       | `crc`         | uint16  | CRC-16/CCITT (CRC field set to 0x0000 during calculation)         |

### Example

```json
{
  "profile": 52,
  "id": "e5f6a7b8c9d0e1f2",
  "mid": 22,
  "ack": true,
  "type": 3,
  "event_code": 3,
  "timestamp": 1622547619000000,
  "altitude": 612.8,
  "velocity": 0.1,
  "rssi": -91,
  "crc": 0
}
```

> **Note:** `"crc": 0` is a placeholder. Implementations must compute the correct CRC-16/CCITT value before transmitting.
