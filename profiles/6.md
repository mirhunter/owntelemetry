# Profile 6 — Air Quality

Indoor and outdoor air quality monitoring. Core fields are CO2 and particulate matter (PM2.5, PM10); VOC, temperature, humidity, AQI, and battery level are optional for sensors that lack those capabilities.

## Packet Types

| Type | Name      | Direction | Description               |
|------|-----------|-----------|---------------------------|
| 1    | `reading` | e->s      | Periodic air quality data |
| 2    | `alert`   | e->s      | Threshold alert           |

## Event Codes

| Code | Name          | Description                                   |
|------|---------------|-----------------------------------------------|
| 1    | `high_co2`    | CO2 concentration exceeded threshold          |
| 2    | `high_pm25`   | PM2.5 concentration exceeded threshold        |
| 3    | `high_pm10`   | PM10 concentration exceeded threshold         |
| 4    | `high_voc`    | VOC concentration exceeded threshold          |
| 5    | `low_battery` | Battery level below operational threshold     |

---

## Type 1 — Reading

| CBOR Key | Field Name      | Type    | Optional | Description                                                    |
|----------|-----------------|---------|----------|----------------------------------------------------------------|
| 0        | `profile`       | uint8   |          | Always `6` for this profile                                    |
| 1        | `id`            | bytes(8)|          | Endpoint device identifier                                     |
| 2        | `mid`           | uint8   |          | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`           | bool    |          | `true` if endpoint requests acknowledgment                     |
| 4        | `type`          | uint8   |          | Always `1` for reading                                         |
| 5        | `timestamp`     | uint32  |          | Unix timestamp in seconds                                      |
| 6        | `co2`           | uint16  |          | CO2 concentration in ppm                                       |
| 7        | `pm25`          | float32 |          | PM2.5 concentration in ug/m3                                   |
| 8        | `pm10`          | float32 |          | PM10 concentration in ug/m3                                    |
| 9        | `voc`           | uint16  | yes      | VOC concentration in ppb                                       |
| 10       | `temperature`   | float32 | yes      | Air temperature in degrees C                                   |
| 11       | `humidity`      | float32 | yes      | Relative humidity as percentage (0.0-100.0)                   |
| 12       | `aqi`           | uint16  | yes      | Computed Air Quality Index (scale and standard are local)      |
| 13       | `battery_level` | float32 | yes      | Battery level as percentage (0.0-100.0)                       |

### Example

```json
{
  "profile": 6,
  "id": "b1e2d3c4f5a6b7c8",
  "mid": 22,
  "ack": false,
  "type": 1,
  "timestamp": 1622547800,
  "co2": 842,
  "pm25": 12.4,
  "pm10": 18.1,
  "voc": 210,
  "temperature": 22.1,
  "humidity": 48.3,
  "aqi": 52,
  "battery_level": 88.0
}
```

## Type 2 — Alert

The `value` field contains the sensor reading that triggered the alert.

| CBOR Key | Field Name    | Type    | Description                                                    |
|----------|---------------|---------|----------------------------------------------------------------|
| 0        | `profile`     | uint8   | Always `6` for this profile                                    |
| 1        | `id`          | bytes(8)| Endpoint device identifier                                     |
| 2        | `mid`         | uint8   | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`         | bool    | `true` if endpoint requests acknowledgment                     |
| 4        | `type`        | uint8   | Always `2` for alert                                           |
| 5        | `event_code`  | uint8   | Event code (see above)                                         |
| 6        | `timestamp`   | uint32  | Unix timestamp in seconds                                      |
| 7        | `value`       | float32 | Sensor value that triggered the alert                          |

### Example

```json
{
  "profile": 6,
  "id": "b1e2d3c4f5a6b7c8",
  "mid": 23,
  "ack": true,
  "type": 2,
  "event_code": 1,
  "timestamp": 1622548200,
  "value": 1450.0
}
```
