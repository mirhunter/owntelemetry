# Profile 7 — Power/Energy Monitor

Single-phase power monitoring for home or small commercial use. Reports real-time voltage, current, power, and cumulative energy consumption. Power factor and frequency are optional for simpler monitors.

## Packet Types

| Type | Name      | Direction | Description                  |
|------|-----------|-----------|------------------------------|
| 1    | `reading` | e->s      | Periodic power and energy data |
| 2    | `alert`   | e->s      | Threshold or state alert     |

## Event Codes

| Code | Name               | Description                                      |
|------|--------------------|--------------------------------------------------|
| 1    | `over_voltage`     | Voltage exceeded upper threshold                 |
| 2    | `under_voltage`    | Voltage fell below lower threshold               |
| 3    | `over_current`     | Current exceeded threshold                       |
| 4    | `high_power`       | Real power exceeded threshold                    |
| 5    | `power_lost`       | Supply power lost (device on backup/battery)     |
| 6    | `power_restored`   | Supply power returned                            |

---

## Type 1 — Reading

`energy` is the cumulative kWh since last reset — servers should store delta between readings to compute consumption over time. `power_factor` and `frequency` are optional for monitors that do not measure them.

| CBOR Key | Field Name     | Type    | Optional | Description                                                    |
|----------|----------------|---------|----------|----------------------------------------------------------------|
| 0        | `profile`      | uint8   |          | Always `7` for this profile                                    |
| 1        | `id`           | bytes(8)|          | Endpoint device identifier                                     |
| 2        | `mid`          | uint8   |          | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`          | bool    |          | `true` if endpoint requests acknowledgment                     |
| 4        | `type`         | uint8   |          | Always `1` for reading                                         |
| 5        | `timestamp`    | uint32  |          | Unix timestamp in seconds                                      |
| 6        | `voltage`      | float32 |          | RMS voltage in V                                               |
| 7        | `current`      | float32 |          | RMS current in A                                               |
| 8        | `power`        | float32 |          | Real power in W                                                |
| 9        | `energy`       | float32 |          | Cumulative energy in kWh since last reset                      |
| 10       | `power_factor` | float32 | yes      | Power factor (0.0-1.0)                                        |
| 11       | `frequency`    | float32 | yes      | Supply frequency in Hz                                         |

### Example

```json
{
  "profile": 7,
  "id": "c2f3e4d5a6b7c8d9",
  "mid": 5,
  "ack": false,
  "type": 1,
  "timestamp": 1622547800,
  "voltage": 241.3,
  "current": 4.82,
  "power": 1063.0,
  "energy": 142.7,
  "power_factor": 0.91,
  "frequency": 50.01
}
```

## Type 2 — Alert

`value` is optional — for `power_lost` and `power_restored` there is no relevant numeric value.

| CBOR Key | Field Name    | Type    | Optional | Description                                                    |
|----------|---------------|---------|----------|----------------------------------------------------------------|
| 0        | `profile`     | uint8   |          | Always `7` for this profile                                    |
| 1        | `id`          | bytes(8)|          | Endpoint device identifier                                     |
| 2        | `mid`         | uint8   |          | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`         | bool    |          | `true` if endpoint requests acknowledgment                     |
| 4        | `type`        | uint8   |          | Always `2` for alert                                           |
| 5        | `event_code`  | uint8   |          | Event code (see above)                                         |
| 6        | `timestamp`   | uint32  |          | Unix timestamp in seconds                                      |
| 7        | `value`       | float32 | yes      | Reading that triggered the alert (omit for power_lost/restored) |

### Example

```json
{
  "profile": 7,
  "id": "c2f3e4d5a6b7c8d9",
  "mid": 6,
  "ack": true,
  "type": 2,
  "event_code": 3,
  "timestamp": 1622548100,
  "value": 18.4
}
```
