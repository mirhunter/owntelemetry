# Profile 51 — Server Health Monitor

> **User-defined profile** (range 51-250). This profile is provided as a reference example of a user-defined profile and is not part of the standard reserved set.

Periodic health snapshots and threshold alerts for a monitored server or VM. Covers CPU, memory, disk, network, and process metrics.

## Packet Types

| Type | Name     | Direction | Description                      |
|------|----------|-----------|----------------------------------|
| 1    | `status` | e->s      | Periodic server health snapshot  |
| 2    | `alert`  | e->s      | Metric threshold alert           |

## Event Codes

Servers derive human-readable names from the code via the profile schema — no string is transmitted on the wire.

| Code | Name                  | Description                          |
|------|-----------------------|--------------------------------------|
| 1    | `high_cpu_load`       | Instant CPU load exceeded threshold  |
| 2    | `high_cpu_temp`       | CPU temperature exceeded threshold   |
| 3    | `high_memory_usage`   | Memory usage exceeded threshold      |
| 4    | `high_disk_usage`     | Disk usage exceeded threshold        |
| 5    | `high_swap_usage`     | Swap usage exceeded threshold        |

---

## Type 1 — Status

| CBOR Key | Field Name      | Type    | Description                                                    |
|----------|-----------------|---------|----------------------------------------------------------------|
| 0        | `profile`       | uint8   | Always `51` for this profile                                   |
| 1        | `id`            | bytes(8)| Endpoint device identifier                                     |
| 2        | `mid`           | uint8   | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`           | bool    | `true` if endpoint requests acknowledgment                     |
| 4        | `type`          | uint8   | Always `1` for status                                          |
| 5        | `timestamp`     | uint32  | Unix timestamp in seconds                                      |
| 6        | `cpu_load`      | float32 | Instant CPU utilisation as percentage (0.0-100.0)             |
| 7        | `cpu_temp`      | float32 | CPU temperature in degrees C                                   |
| 8        | `load_1`        | float32 | 1-minute load average                                          |
| 9        | `load_5`        | float32 | 5-minute load average                                          |
| 10       | `load_15`       | float32 | 15-minute load average                                         |
| 11       | `memory_used`   | float32 | Memory used as percentage (0.0-100.0)                         |
| 12       | `swap_used`     | float32 | Swap used as percentage (0.0-100.0)                           |
| 13       | `disk_used`     | float32 | Root disk used as percentage (0.0-100.0)                      |
| 14       | `net_rx`        | uint32  | Network receive rate in KB/s                                   |
| 15       | `net_tx`        | uint32  | Network transmit rate in KB/s                                  |
| 16       | `uptime`        | uint32  | System uptime in seconds                                       |
| 17       | `processes`     | uint16  | Number of running processes                                    |

### Example

```json
{
  "profile": 51,
  "id": "d4a5b6c7e8f9a0b1",
  "mid": 3,
  "ack": false,
  "type": 1,
  "timestamp": 1622547800,
  "cpu_load": 23.4,
  "cpu_temp": 51.0,
  "load_1": 0.82,
  "load_5": 0.74,
  "load_15": 0.61,
  "memory_used": 61.2,
  "swap_used": 4.1,
  "disk_used": 43.7,
  "net_rx": 1240,
  "net_tx": 380,
  "uptime": 1728000,
  "processes": 214
}
```

## Type 2 — Alert

The `value` field contains the metric reading that triggered the alert.

| CBOR Key | Field Name    | Type      | Description                                                    |
|----------|---------------|-----------|----------------------------------------------------------------|
| 0        | `profile`     | uint8     | Always `51` for this profile                                   |
| 1        | `id`          | bytes (8) | Endpoint device identifier                                     |
| 2        | `mid`         | uint8     | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`         | bool      | `true` if endpoint requests acknowledgment                     |
| 4        | `type`        | uint8     | Always `2` for alert                                           |
| 5        | `event_code`  | uint8     | Event code (see above)                                         |
| 6        | `timestamp`   | uint32    | Unix timestamp in seconds                                      |
| 7        | `value`       | float32   | Metric value that triggered the alert                          |

### Example

```json
{
  "profile": 51,
  "id": "d4a5b6c7e8f9a0b1",
  "mid": 4,
  "ack": true,
  "type": 2,
  "event_code": 3,
  "timestamp": 1622548400,
  "value": 94.7
}
```
