# Profile 9 — Soil/Agricultural Sensor

Soil condition monitoring for agricultural and horticultural applications. Core fields are soil moisture and soil temperature. Electrical conductivity (EC), pH, ambient conditions, and light level are optional for sensors that include those measurements.

## Packet Types

| Type | Name      | Direction | Description              |
|------|-----------|-----------|--------------------------|
| 1    | `reading` | e->s      | Periodic soil conditions |
| 2    | `alert`   | e->s      | Threshold alert          |

## Event Codes

| Code | Name                  | Description                                    |
|------|-----------------------|------------------------------------------------|
| 1    | `low_soil_moisture`   | Soil moisture fell below lower threshold       |
| 2    | `high_soil_moisture`  | Soil moisture exceeded upper threshold         |
| 3    | `low_temperature`     | Soil temperature fell below lower threshold    |
| 4    | `high_temperature`    | Soil temperature exceeded upper threshold      |
| 5    | `low_battery`         | Battery level below operational threshold      |

---

## Type 1 — Reading

`soil_moisture` and `soil_temperature` are the required core measurements. All other fields are optional — omit those not supported by the hardware.

| CBOR Key | Field Name            | Type    | Optional | Description                                                    |
|----------|-----------------------|---------|----------|----------------------------------------------------------------|
| 0        | `profile`             | uint8   |          | Always `9` for this profile                                    |
| 1        | `id`                  | bytes(8)|          | Endpoint device identifier                                     |
| 2        | `mid`                 | uint8   |          | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`                 | bool    |          | `true` if endpoint requests acknowledgment                     |
| 4        | `type`                | uint8   |          | Always `1` for reading                                         |
| 5        | `timestamp`           | uint32  |          | Unix timestamp in seconds                                      |
| 6        | `soil_moisture`       | float32 |          | Volumetric water content as percentage (0.0-100.0)            |
| 7        | `soil_temperature`    | float32 |          | Soil temperature in degrees C                                  |
| 8        | `soil_ec`             | float32 | yes      | Soil electrical conductivity in mS/cm                          |
| 9        | `soil_ph`             | float32 | yes      | Soil pH (0.0-14.0)                                            |
| 10       | `ambient_temperature` | float32 | yes      | Ambient air temperature in degrees C                           |
| 11       | `ambient_humidity`    | float32 | yes      | Ambient relative humidity as percentage (0.0-100.0)           |
| 12       | `lux`                 | float32 | yes      | Light level in lux                                             |
| 13       | `battery_level`       | float32 | yes      | Battery level as percentage (0.0-100.0)                       |

### Example

```json
{
  "profile": 9,
  "id": "e5f6a7b8c9d0e1f2",
  "mid": 8,
  "ack": false,
  "type": 1,
  "timestamp": 1622547800,
  "soil_moisture": 34.2,
  "soil_temperature": 18.6,
  "soil_ec": 0.42,
  "soil_ph": 6.8,
  "ambient_temperature": 21.0,
  "ambient_humidity": 65.0,
  "lux": 28400.0,
  "battery_level": 91.0
}
```

## Type 2 — Alert

The `value` field contains the sensor reading that triggered the alert.

| CBOR Key | Field Name    | Type    | Description                                                    |
|----------|---------------|---------|----------------------------------------------------------------|
| 0        | `profile`     | uint8   | Always `9` for this profile                                    |
| 1        | `id`          | bytes(8)| Endpoint device identifier                                     |
| 2        | `mid`         | uint8   | Rolling message ID; server deduplicates on `(mid, timestamp)` |
| 3        | `ack`         | bool    | `true` if endpoint requests acknowledgment                     |
| 4        | `type`        | uint8   | Always `2` for alert                                           |
| 5        | `event_code`  | uint8   | Event code (see above)                                         |
| 6        | `timestamp`   | uint32  | Unix timestamp in seconds                                      |
| 7        | `value`       | float32 | Sensor value that triggered the alert                          |

### Example

```json
{
  "profile": 9,
  "id": "e5f6a7b8c9d0e1f2",
  "mid": 9,
  "ack": true,
  "type": 2,
  "event_code": 1,
  "timestamp": 1622548400,
  "value": 18.1
}
```
